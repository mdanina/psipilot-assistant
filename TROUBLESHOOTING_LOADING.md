# Решение проблемы: Приложение не загружается

## Шаг 1: Проверьте консоль браузера

1. Откройте DevTools (F12)
2. Перейдите на вкладку **Console**
3. Посмотрите на **красные ошибки**
4. Скопируйте все ошибки

**Частые ошибки:**
- `Failed to fetch` - проблема с Supabase URL
- `Missing VITE_SUPABASE_URL` - не настроены переменные окружения
- `Cannot find module` - проблема с импортами
- `Uncaught Error` - ошибка в коде

## Шаг 2: Проверьте dev сервер

Убедитесь, что dev сервер запущен:

```bash
npm run dev
```

Должно быть сообщение:
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:3000/
  ➜  Network: use --host to expose
```

## Шаг 3: Проверьте переменные окружения

Убедитесь, что файл `.env.local` существует и содержит:

```env
VITE_SUPABASE_URL=https://lulebraggalast.beget.app
VITE_SUPABASE_ANON_KEY=ваш-ключ
VITE_ENCRYPTION_KEY=ваш-ключ-шифрования
```

**Важно:** После изменения `.env.local` нужно **перезапустить** dev сервер!

## Шаг 4: Очистите кэш

### Очистка кэша Vite:
```bash
# Остановите dev сервер (Ctrl+C)
# Удалите кэш
Remove-Item -Recurse -Force node_modules\.vite -ErrorAction SilentlyContinue
# Перезапустите
npm run dev
```

### Очистка кэша браузера:
- Нажмите **Ctrl+Shift+R** (жесткая перезагрузка)
- Или откройте в режиме инкогнито

## Шаг 5: Проверьте порт

Убедитесь, что порт 3000 свободен:

```bash
netstat -ano | findstr :3000
```

Если порт занят, либо освободите его, либо измените порт в `vite.config.ts`.

## Шаг 6: Переустановите зависимости

Если после git pull появились новые зависимости:

```bash
npm install
```

## Шаг 7: Проверьте логи терминала

Посмотрите на вывод в терминале, где запущен `npm run dev`. 
Там могут быть ошибки компиляции или предупреждения.

## Что делать дальше?

1. **Скопируйте ошибки из консоли браузера (F12 → Console)**
2. **Проверьте, что dev сервер запущен и работает**
3. **Убедитесь, что `.env.local` настроен правильно**
4. **Попробуйте очистить кэш и перезапустить**

Если проблема не решается, пришлите:
- Ошибки из консоли браузера
- Вывод терминала с `npm run dev`
- Содержимое `.env.local` (без реальных ключей, только структуру)
